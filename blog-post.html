Building a 3D Particle System from Scratch

A technical deep dive into creating a performant particle system using Three.js, simplex noise, and GPU-friendly techniques.

By Pintan — 12 min read

Particle systems are one of those things that look magical but are surprisingly straightforward once you understand the core concepts. In this post, I'll walk through how I built the 3D particle universe for my Cosmic Sandbox project.

The Foundation: BufferGeometry

Three.js provides BufferGeometry as the backbone for rendering large numbers of vertices efficiently. Unlike traditional geometry, BufferGeometry stores data in typed arrays that can be sent directly to the GPU with minimal overhead.

For a particle system, we need at minimum a position attribute (3 floats per particle: x, y, z) and optionally a color attribute. With 30,000 particles, that's 90,000 floats for positions alone — a trivially small amount for modern GPUs.

Simplex Noise for Organic Movement

To make the particles move organically, I implemented 3D simplex noise. Unlike Perlin noise, simplex noise has a lower computational complexity and produces fewer directional artifacts, making it ideal for fluid-looking animations.

The key insight is to use time as the fourth dimension input to the noise function. This creates continuously evolving movement patterns that never repeat exactly.

Simplex noise gives us natural-looking motion without the overhead of physics simulations. It's the perfect tool for creating living, breathing particle systems.

Performance Considerations

With 30,000 particles, every optimization matters. Here are the key strategies I used:

First, avoid allocations in the render loop. All temporary vectors and colors are pre-allocated once and reused every frame. This prevents garbage collection pauses that would destroy the smoothness of the animation.

Second, use AdditiveBlending for particles. This not only looks better (bright particles glow against dark backgrounds), but also reduces overdraw issues since particles don't obscure each other.

Third, compute target positions in a separate buffer and interpolate. This decouples the logical simulation from the visual state, allowing for smooth transitions between different modes.

Putting It All Together

The final system supports multiple modes: a cosmic solar system with orbiting planets, preset celestial patterns, custom image-based 3D reliefs, and built-in geometric shapes. Each mode simply writes to the same target buffer, and the animation loop handles the interpolation.

The result is a flexible, performant particle engine that can power everything from interactive art to data visualization.
